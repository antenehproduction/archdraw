Schema Mapping
</h1>

<ul>
<li><a href="#h_01H08PR7CKBH246NWR0N2TNM0J" target="_self">Mapping Columns with Selector</a></li>

<p>When making data updates, you may encounter column mismatch errors which are now handled with the Schema Mapping tool.</p>
<p><img src="https://support.socrata.com/hc/article_attachments/360028349033/mceclip0.png" width="701" height="327" /></p>
<h2 id="h_01H08PR7CKBH246NWR0N2TNM0J">Mapping Columns with Selector</h2>
<p>From this tool, you can use the <strong>New Mapping</strong> dropdown menu to select the source file column that will be mapped to the published data.</p>
<p><img src="https://support.socrata.com/hc/article_attachments/360027519574/mceclip1.png" width="714" height="314" /></p>
<h2 id="h_01H08PRGJZ6J1XSVZ8FGT7AKE8">Mapping Columns with Data Transforms</h2>
<p>If the published dataset has a column that is created through transforms, such as a synthetic location column. The kabob menu next to the source file column selector will allow for specifying a transform to create the column mapping.</p>
<p class="wysiwyg-text-align-center"><img src="https://support.socrata.com/hc/article_attachments/360028349353/mceclip2.png" width="693" height="119" /></p>
<h2 id="h_01H08PRPTCESS6M59K3ZBK8VC7" class="wysiwyg-text-align-left">Saving Column Mapping Changes</h2>
<p class="wysiwyg-text-align-left">Once all columns have been mapped, the tool will unlock the option to <strong>Save and Review Data</strong>.</p>
<p class="wysiwyg-text-align-left"><img src="https://support.socrata.com/hc/article_attachments/360028349433/mceclip3.png" width="715" height="364" /></p>
<p class="wysiwyg-text-align-left">After saving, the publishing tool will redirect to the data table preview where you can review the data that is now mapped to their appropriate columns!</p>
